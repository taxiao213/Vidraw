# Excalidraw 屏幕录制功能

## 功能概述

Excalidraw 新增了屏幕录制功能，允许用户录制画布上的任意区域，支持自定义宽高比和分辨率，并可添加麦克风音频。

## 功能特性

### 1. 区域选择

- **拖拽选择框**：点击录制按钮后，可通过拖拽选择需要录制的区域
- **调整大小**：拖拽四个角的控制点可调整录制区域大小
- **移动位置**：拖拽选择框内部可移动录制区域位置
- **点击外部取消**：点击选择框外的暗色区域可取消选择

### 2. 宽高比设置

支持多种预设宽高比：

| 选项 | 宽高比 | 适用场景   |
| ---- | ------ | ---------- |
| Free | 自由   | 任意比例   |
| 9:16 | 0.5625 | 竖屏短视频 |
| 3:4  | 0.75   | 竖屏内容   |
| 1:1  | 1      | 正方形视频 |
| 4:3  | 1.333  | 传统视频   |
| 16:9 | 1.778  | 宽屏视频   |

### 3. 分辨率设置

支持多种输出分辨率：

| 选项            | 分辨率    | 说明                       |
| --------------- | --------- | -------------------------- |
| Original        | 原始尺寸  | 保持选择区域的原始像素大小 |
| 480p (SD)       | 854×480   | 标清，文件较小             |
| 720p (HD)       | 1280×720  | 高清，平衡画质与文件大小   |
| 1080p (Full HD) | 1920×1080 | 全高清，推荐使用           |
| 2K (QHD)        | 2560×1440 | 超高清                     |
| 4K (UHD)        | 3840×2160 | 超高清，最大画质           |

### 4. 音频录制

- **麦克风支持**：自动请求麦克风权限，录制系统音频输入
- **自动检测**：支持麦克风时自动添加音轨
- **编解码器**：使用 Opus 音频编解码器确保音质

### 5. 视频输出

- **格式转换**：自动将 WebM 转换为 MP4 格式
- **文件保存**：支持系统文件选择器保存，或自动下载
- **进度显示**：转换过程中显示进度条

## 使用方法

### 开始录制

1. 点击工具栏中的录制按钮（摄像头图标）
2. 在屏幕上拖拽选择录制区域
3. 选择所需的宽高比和分辨率
4. 点击「Start Recording」按钮

### 录制过程

- 浏览器会弹出屏幕共享选择对话框
- 选择要共享的屏幕、窗口或标签页
- 建议选择「当前标签页」以获得最佳效果
- 录制时会在录制区域四角显示红色边框提示（不会被录制进视频）

### 停止录制

1. 点击右上角的停止按钮
2. 选择保存位置和文件名
3. 等待视频转换完成
4. 视频将自动保存到指定位置

### 取消录制

- 选择区域时：按 `Esc` 键或点击选择框外部
- 录制过程中：点击停止按钮旁边的取消选项

## 技术实现

### 核心技术

- **MediaRecorder API**：用于捕获屏幕和音频
- **Canvas API**：用于裁剪和缩放视频帧
- **FFmpeg.wasm**：用于 WebM 到 MP4 的格式转换

### 文件结构

```
excalidraw-app/
├── components/
│   ├── RecordingOverlay.tsx    # 区域选择 UI
│   ├── RecordingOverlay.scss   # 样式文件
│   └── RecordingProgress.tsx   # 录制进度 UI
├── recording/
│   ├── recordingState.ts       # 状态管理
│   ├── useRecording.ts         # 录制逻辑 Hook
│   └── ffmpegService.ts        # FFmpeg 转换服务
└── camera/
    ├── cameraState.ts          # 摄像头状态
    └── useCamera.ts            # 摄像头逻辑
```

### 状态流转

```
idle → selecting → ready → recording → converting → idle
                ↘ (cancel) ↗
```

## 系统启动指南

### 环境要求

- Node.js >= 18.0.0
- Yarn >= 1.22.0
- 现代浏览器（Chrome、Firefox、Edge 等）

### 安装依赖

```bash
yarn install
```

### 启动开发服务器

```bash
yarn start
```

服务器启动后，访问控制台显示的本地地址（如 http://localhost:3000/）

## 注意事项

1. **权限请求**：

   - 首次使用需要授予屏幕共享权限
   - 麦克风录制需要额外授权

2. **性能建议**：

   - 4K 录制对设备性能要求较高
   - 长时间录制会占用较多内存
   - 建议选择与目标平台匹配的分辨率

3. **已知限制**：
   - 系统音频录制依赖浏览器支持
   - 移动端支持有限
